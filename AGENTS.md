## 目的と適用範囲
- このファイルが置かれたディレクトリ以下に適用されます。

- 迷ったらリポジトリの README や CONTRIBUTING を確認します。

## 作業フロー
- 変更は小さく、目的に沿った内容に限定します。

- コミットは論理的な単位で分け、説明的なメッセージを付けます。

## コーディングとスタイル
- 既存のコーディング規約・命名・構成に合わせます。
- 依存関係の追加は必要最小限にし、理由を明記します。
- 既存の機能や API を壊さないように注意します。

## テストと検証

## ドキュメント

- 変更点は簡潔で利用者目線の説明を心がけます。

## コミュニケーション
- 最終報告では変更概要、テスト結果、注意点を簡潔にまとめます。
- 追加のフォローアップが必要なら明示します。
